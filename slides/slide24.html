Erreurs fréquentes

Les pièges classiques à éviter

Oublier git add

Le fichier reste "Untracked" ou "Modified" et n'est pas inclus dans le commit.

Confondre Commit & Push

Le commit sauvegarde en local seulement. Sans push, GitHub ne voit rien.

Travailler sur main

Risqué ! Si ça plante, c'est toute la version stable du projet qui est cassée.

Messages de commit vagues

"Fix", "Update"... Impossible de savoir ce qui a changé en relisant l'historique.

Pusher des fichiers sensibles

Mots de passe, clés API ou données privées publiés publiquement par erreur.

Ignorer .gitignore

Pollution du dépôt avec des fichiers système, logs ou dossiers de compilation.

Solution ultime :
git status

à utiliser avant chaque commande !
