Bonnes pratiques

Les règles d'or pour un dépôt propre

Commits clairs

Utilisez l'impératif. Ex: "Ajoute formulaire login"

Commits petits et fréquents

Ne committez pas 3 jours de travail d'un coup.

Une branche = Une fonctionnalité

Séparez votre travail pour faciliter la relecture.

Ne jamais push sur main directement

Utilisez toujours des Pull Requests.

Jamais de mots de passe

Les secrets (API Keys, MDP) ne vont jamais sur GitHub.

La Liste Noire

Ce fichier dit à Git quels fichiers ignorer. Indispensable pour ne pas polluer votre repo avec des fichiers temporaires ou sensibles.
