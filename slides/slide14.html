Travail en équipe

Le flux Pull Request

1

FORK

Copie le repo distant sur ton propre compte GitHub.

2

BRANCH

Crée une branche locale pour ta nouvelle fonctionnalité.

3

COMMIT

Fais tes modifications et sauvegarde-les localement.

4

PUSH

Envoie ta branche et tes commits sur GitHub.

5

PULL REQUEST

Demande à intégrer ton code dans le projet original.

6

CODE REVIEW

Les collègues vérifient, commentent et valident.

7

MERGE

C'est accepté ! Ton code rejoint la branche principale.

Chacun travaille sans gêner les autres
