Démo pratique - Partie 3

Live : Pull Request complète

1

Créer branche

git switch -c feature

Crée une branche dédiée pour isoler le travail.

2

Modifier

vim README.md

Apporte tes modifications au fichier.

3

Sauvegarder

git add & commit

Prépare et enregistre les changements localement.

4

Envoyer

git push -u origin

Publie ta branche sur le dépôt GitHub.

5

Ouvrir PR

Sur GitHub.com

Crée la Pull Request depuis l'interface web.

6

Merger

Bouton "Merge"

Valide et fusionne le code dans la branche principale.

7

Retour main

git switch main

Reviens sur la branche principale locale.

8

À jour

git pull

Récupère la version fusionnée sur ton PC.

C'EST LE WORKFLOW QUOTIDIEN !
